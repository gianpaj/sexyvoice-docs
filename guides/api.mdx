Use the API when you want to generate speech from your own backend or app.

If you prefer a terminal workflow, use the official CLI:

- Repo: [`gianpaj/sexyvoice-cli`](https://github.com/gianpaj/sexyvoice-cli)
- Good for browser-based login, listing voices, generating speech, and producing
  copy-pasteable `curl` commands

## Prerequisites

- A SexyVoice account
- API key from **Dashboard -> API Keys**
- Paid account

## 1. Authenticate

Send your API key as a Bearer token:

```bash
-H 'Authorization: Bearer sk_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx'
```

### CLI

```bash
sexyvoice login
```

## 2. Generate Speech

### Using Gemini (gpro)

Gemini voices support the freeform `style` parameter and optional `seed`.

```bash
curl -X POST 'https://sexyvoice.ai/api/v1/speech' \
  -H 'Authorization: Bearer sk_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx' \
  -H 'Content-Type: application/json' \
  -d '{
    "model": "gpro",
    "voice": "poe",
    "input": "Hello from SexyVoice API",
    "style": "calm",
    "seed": 1234
  }'
```

### Using Grok (xAI)

Grok voices support both MP3 (default) and WAV output via `response_format`.
They ignore the `style` field. Use expressive tags directly in `input`
instead, such as `[laugh]` or `<whisper>...</whisper>`.

```bash
curl -X POST 'https://sexyvoice.ai/api/v1/speech' \
  -H 'Authorization: Bearer sk_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx' \
  -H 'Content-Type: application/json' \
  -d '{
    "model": "grok",
    "voice": "eve",
    "input": "Hello from Grok! [laugh] This is exciting.",
    "response_format": "mp3"
  }'
```

Successful response includes:

- `url`: generated audio URL
- `credits_used`
- `credits_remaining`
- `usage`

### CLI

```bash
sexyvoice tts "Hello from SexyVoice API" --voice kore --model-id gpro -o hello.wav
```

## Voice Capabilities

Use `GET /api/v1/voices` to inspect the available voices for each model.

- `formats` lists the supported output formats for that voice
- `supports_style` tells you whether the voice accepts the freeform `style`
  parameter
- Today, Gemini (`gpro`) voices return `supports_style: true`
- Grok (`grok`) and standard (`orpheus`) voices return `supports_style: false`

### CLI

```bash
sexyvoice voices
```

The CLI also lets you skip `--model-id` for `tts` when the selected voice
uniquely resolves through `GET /api/v1/voices`.

## 3. Check Balance

```bash
curl -X GET 'https://sexyvoice.ai/api/v1/billing' \
  -H 'Authorization: Bearer sk_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx'
```

Response includes:

- `creditsLeft`
- `lastUpdated`
- `userId`
- `lastBillingTransaction`

## 4. Handle Errors + Debugging

- Every response includes a `request-id` header. Log it in your app.
- On failures, inspect `error.code` and `error.type`.
- See [Error Codes](/api-reference/errors) for the full list.

## Useful Endpoints

- `POST /api/v1/speech`
- `GET /api/v1/billing`
- `GET /api/v1/voices`
- `GET /api/v1/models`
- `GET /api/v1/openapi`
